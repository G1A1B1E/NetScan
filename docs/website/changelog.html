Versioning Policy

NetScan follows Semantic Versioning:

  • MAJOR (X.0.0) ‚Äî Incompatible API changes
  • MINOR (0.X.0) ‚Äî New features, backwards compatible
  • PATCH (0.0.X) ‚Äî Bug fixes, backwards compatible

Release Schedule

  • Major releases: As needed for significant changes
  • Minor releases: Monthly or bi-monthly
  • Patch releases: As needed for critical fixes

Deprecation Policy

Deprecated features will be marked with warnings for at least one minor version before removal.
